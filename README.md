## Описание проекта

Данный проект разработан в рамках закрепления навыков по объектно-ориентированному программированию, взаимодействию со стандартной библиотекой шаблонов (STL), обработке потоков файлового ввода-вывода, организации модульной архитектуры на C++ и проектированию консольных интерфейсов с валидацией пользовательского ввода.

## Основная цель

Создание полноценного настольного CRUD-приложения, обеспечивающего учет регулярных платежей и подписок. В систему заложены механизмы сортировки, фильтрации и поиска записей, а также вычисление финансовых показателей (анализ затрат, прогнозирование платежей на неделю вперед, определение наиболее затратных категорий).

## Функционал приложения

### Управление подписками (CRUD)
- Создание новой записи с автоматическим назначением уникального ID.
- Корректировка параметров существующих подписок.
- Удаление подписок (выборочно: по уникальному идентификатору либо названию сервиса).
- Контроль корректности вводимых данных (диапазоны цен, формат даты, валидация пустых строк).

### Поиск и выборка данных
- Поиск записей по имени сервиса (с поддержкой частичного совпадения и игнорированием регистра).
- Фильтрация по закрепленным категориям («Развлечения», «Софт», «Обучение», «Коммунальные», «Другое»).
- Фильтрация по заданному ценовому коридору.
- Выборка предстоящих транзакций в границах временного интервала.

### Инструменты сортировки
- По месячной стоимости (в порядке возрастания или убывания).
- По дате планируемой оплаты.
- По алфавиту (наименование услуги).

### Аналитические инструменты
- Суммирование расходов по всем действующим подпискам (приведение стоимости к ежемесячному эквиваленту).
- Отслеживание расходов, запланированных на ближайшие 7 суток.
- Ранжирование топ-3 категорий по объему финансовых затрат.

### Файловая система
- Автоматический импорт сохраненных данных из `data/subscriptions.txt` при старте.
- Сохранение актуального состояния базы данных вручную по запросу пользователя.
- Структура строки хранения данных: `id, service_name, category, monthly_cost, billing_cycle, next_payment_date, status`.

## Инструкция по сборке и запуску

### Предварительные системные требования

- Компилятор с поддержкой стандарта C++17 (MSVC от версии 2017, GCC от версии 7, Clang от версии 5).
- Утилита автоматизации сборки CMake версии 3.10 и выше.
- Git-клиент (для получения исходного кода).

### Способ 1. Автоматизированная сборка через CMake

```bash
# Получение копии репозитория
git clone https://github.com/Reutop/SubscriptionTracker.git
cd SubscriptionTracker

# Подготовка окружения для компиляции
mkdir build && cd build

# Создание конфигурационных файлов сборщика
cmake ..

# Компиляция исполняемого файла
cmake --build .

# Старт программы на платформе Windows
cd Debug && SubscriptionTracker.exe

# Старт программы на UNIX-подобных системах (Linux/macOS)
./SubscriptionTracker
```

### Способ 2. Ручная компиляция средствами компилятора

```bash
# Получение копии репозитория
git clone https://github.com/Reutop/SubscriptionTracker.git
cd SubscriptionTracker

# Компиляция всех компонентов проекта напрямую в бинарный файл
g++ -std=c++17 src/*.cpp -o SubscriptionTracker

# Запуск собранного приложения
./SubscriptionTracker
```

### Способ 3. Интегрированная среда разработки MS Visual Studio

1. Запустите MS Visual Studio.
2. Перейдите по пути: *Файл -> Создать -> Проект* и выберите шаблон *Консольное приложение C++*.
3. Установите название проекта как `SubscriptionTracker`.
4. Перенесите исходные файлы из директории `src/` в созданный проект:
   - Через контекстное меню пункта *«Файлы исходного кода»* импортируйте все `.cpp`-файлы.
   - Через контекстное меню пункта *«Заголовочные файлы»* импортируйте все `.h`-файлы.
5. Настройте конфигурацию сборки проекта:
   - В разделе *C/C++ -> Язык* установите параметр *Стандарт языка C++* на значение `ISO C++17 Standard (/std:c++17)`.
   - В разделе *C/C++ -> Препроцессор* добавьте в список определений директиву `_CRT_SECURE_NO_WARNINGS`.
   - В разделе *C/C++ -> Командная строка* укажите дополнительный параметр `/utf-8`.
   - В разделе *Отладка* установите значение *Рабочий каталог* как `$(ProjectDir)`.
6. Выполните компиляцию проекта (сочетанием клавиш `Ctrl+Shift+B`) и запустите отладку (`F5`).

> **Обратите внимание:** Если при первом запуске приложения каталог `data/` отсутствует, программа автоматически создаст необходимую папку и сгенерирует файл `subscriptions.txt` с демонстрационным набором данных.
